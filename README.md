# acnh.github.io

Гайды чата ACNH (статический сайт для GitHub Pages).

## Как открыть сайт локально

В проекте нет сборки — это чистый HTML/CSS.

**Самый простой способ:**
1. Откройте файл `index.html` двойным кликом в браузере.

**Если нужен локальный сервер (рекомендуется для корректных относительных ссылок):**
1. Запустите любой статический сервер в корне репозитория.
2. Откройте `http://localhost:PORT`.

Примеры (любой из вариантов):
```bash
# Python
python -m http.server 8000

# Node
npx serve .
```

## Структура проекта

- `index.html` — главная страница со списком гайдов и ссылками.
- `terminology/index.html` — страница терминов (пример перенесённого гайда).
- `rules/index.html` — правила этикета.
- `newspaper/index.html` — газета чата (выпуски).
- `contest/index.html` — конкурсы чата.
- `part_one/index.html` — основные вопросы про жителей.
- `residents_gifts/index.html` — жители и подарки.
- `styles.css` — общий стиль для всех страниц.
- `assets/` — папка с локальными изображениями.
- `components/` — переиспользуемые компоненты (шапка, подвал).

## Как добавлять новые страницы гайдов

1. Создайте папку для новой страницы (например, `rules/`) и добавьте в нее `index.html`.
2. Используйте структуру из существующих страниц, но вместо копирования шапки/подвала используйте плейсхолдеры:
   ```html
   <body>
     <div id="header-placeholder"></div>
     
     <main id="content">
       <!-- Ваш контент -->
     </main>
     
     <div id="footer-placeholder"></div>
     <script src="../components/load-components.js"></script>
   </body>
   ```
3. Подключите `../styles.css` (или `styles.css` для главной страницы).
4. Добавьте ссылку на новую страницу в `index.html` (например, `rules/`).

**Важно:** Шапка и подвал загружаются автоматически через скрипт `load-components.js`. Не копируйте их вручную — используйте плейсхолдеры!

## Примечания по коду

- Сайт полностью статический, минимальный JS используется только для загрузки общих компонентов (шапка/подвал).
- Изображения хранятся локально в папке `assets/` для независимости от внешних CDN.
- Можно переносить контент из Tilda блоками и упрощать разметку.

## Структура компонентов

- `components/header.html` — общая шапка сайта (навигация, логотип).
- `components/footer.html` — общий подвал сайта (ссылки, копирайт).
- `components/load-components.js` — скрипт для автоматической загрузки шапки и подвала на всех страницах.

При добавлении новых страниц используйте плейсхолдеры `<div id="header-placeholder"></div>` и `<div id="footer-placeholder"></div>` вместо копирования шапки/подвала.
