# acnh.github.io

Гайды чата ACNH (статический сайт для GitHub Pages).

## Как открыть сайт локально

В проекте нет сборки — это чистый HTML/CSS.

**Самый простой способ:**
1. Откройте файл `index.html` двойным кликом в браузере.

**Если нужен локальный сервер (рекомендуется для корректных относительных ссылок):**
1. Запустите любой статический сервер в корне репозитория.
2. Откройте `http://localhost:PORT`.

Примеры (любой из вариантов):
```bash
# Python
python -m http.server 8000

# Node
npx serve .
```

## Структура проекта

- `index.html` — главная страница со списком гайдов и ссылками.
- `template.html` — **шаблон для создания новых страниц** (используйте его как основу).
- `chat/` — страницы про чат и активность:
  - `chat/terminology/index.html` — страница терминов.
  - `chat/rules/index.html` — правила этикета.
  - `chat/newspaper/index.html` — газета чата (выпуски).
  - `chat/contest/index.html` — конкурсы чата.
- `residents/` — страницы про жителей:
  - `residents/part_one/index.html` — основные вопросы про жителей.
  - `residents/residents_gifts/index.html` — жители и подарки.
- `styles.css` — общий стиль для всех страниц (унифицированный, с комментариями).
- `assets/` — папка с локальными изображениями.
- `components/` — переиспользуемые компоненты (шапка, подвал).

## Как добавлять новые страницы гайдов

### Быстрый способ (рекомендуется)

1. Определите, к какому блоку относится новая страница:
   - `chat/` — для страниц про чат и активность
   - `residents/` — для страниц про жителей
   - Или создайте новую папку для нового блока
2. Скопируйте файл `template.html` в соответствующую папку и переименуйте в `index.html`.
3. Заполните шаблон:
   - Измените `<title>` и `<meta name="description">` в `<head>`.
   - Обновите заголовок и описание в hero-секции.
   - Добавьте свой контент в секции.
   - **Важно:** Используйте абсолютные пути от корня (`/`):
     - `href="/styles.css"` для стилей
     - `src="/components/load-components.js"` для скрипта
     - `src="/assets/..."` для изображений
     - Это работает одинаково из любой папки и не зависит от глубины вложенности
4. Добавьте ссылку на новую страницу в `index.html` (например, `chat/new-page/`).

### Варианты hero-секций

Шаблон поддерживает несколько вариантов hero-секций:

**1. Стандартная hero с изображением:**
```html
<section class="hero" style="background-image: url('../assets/path/to/image.jpg');">
```

**2. Hero с цветным фоном (компактная):**
```html
<section class="hero hero-compact hero-bg-qa">
```
Доступные варианты: `hero-bg-newspaper`, `hero-bg-contest`, `hero-bg-qa`, `hero-bg-terminology`

**3. Hero по центру:**
```html
<section class="hero hero-compact hero-centered hero-bg-terminology">
```

### Важные замечания

- Шапка и подвал загружаются автоматически через скрипт `load-components.js`. Не копируйте их вручную — используйте плейсхолдеры!
- Путь к `styles.css` должен быть `../styles.css` для страниц в подпапках.
- Путь к `load-components.js` должен быть `../components/load-components.js` для страниц в подпапках.

## Примечания по коду

- Сайт полностью статический, минимальный JS используется только для загрузки общих компонентов (шапка/подвал).
- Изображения хранятся локально в папке `assets/` для независимости от внешних CDN.
- Можно переносить контент из Tilda блоками и упрощать разметку.

   - Единая система именования классов
   - Переиспользуемые компоненты стилей
   - Готовые варианты для разных типов страниц (Q&A, терминология, конкурсы и т.д.)

- Для Q&A страниц используйте классы `qa-card`, `qa-answer` и т.д.

## Структура компонентов

- `components/header.html` — общая шапка сайта (навигация, логотип).
- `components/footer.html` — общий подвал сайта (ссылки, копирайт).
- `components/load-components.js` — скрипт для автоматической загрузки шапки и подвала на всех страницах.

При добавлении новых страниц используйте плейсхолдеры `<div id="header-placeholder"></div>` и `<div id="footer-placeholder"></div>` вместо копирования шапки/подвала.
